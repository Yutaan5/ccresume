# ccresume

Claude Code の `/resume` に出てくる過去セッションをブラウズ・操作する macOS 向け TUI。

```
go build -o ccresume .
./ccresume
```

`~/.claude/projects/` 配下のセッション（JSONL）を読み取り、ディレクトリ別に一覧表示する。
右ペインには選択中セッションの**最後のアシスタント返答**を Markdown レンダリングして表示する。
UUID 形式ではないファイル名はセッションとして読み込まない。

## キー操作

| キー | 動作 |
|---|---|
| `j` / `k` / `↑↓` | セッション選択 |
| `enter` | 呼び出し（resume） |
| `d` / `x` | 削除（`y/N` 確認あり） |
| `tab` / `p` | ディレクトリ切り替え |
| `/` | インクリメンタル検索（日本語可） |
| `J` / `K`, `ctrl+d` / `ctrl+u` | プレビューのスクロール |
| `r` | 再スキャン |
| `q` / `esc` | 終了 |

## 呼び出しの挙動

- **cmux 内**: 現在のワークスペースに新しいターミナルタブを作り、そこで
  `cd <セッションの作業ディレクトリ> && claude --resume <id>` を実行する
  （失敗時は `cmux new-workspace` にフォールバック）。
- **通常のターミナル**: TUI を終了し、同じターミナルで作業ディレクトリに
  移動してから `claude --resume <id>` に置き換わる（exec）。
- 保存された作業ディレクトリが空、消失済み、または利用不能な場合は resume を中止する。

## 削除の挙動

セッションの `.jsonl` ファイルのみを完全削除する（ゴミ箱には入らない）。
同名 UUID のサブディレクトリ（サブエージェントの作業データ）は削除しない。

## デバッグ

`CCRESUME_DEBUG_KEYS=1 ./ccresume` でフッターに受信キーを表示する。
